# RingoTrack 用户故事流

## 1）新用户第一次打开 RingoTrack

### 场景
用户安装后第一次启动 App。

### 步骤
1. 用户安装后第一次启动 App
2. 看到一个「欢迎页 / 空状态」：中间是一堵空空的小绿砖墙轮廓，配一句话：
   - 「还没有记录到任何绘画时间，打开你常用的绘画软件，RingoTrack 会在这里堆起你的画画小绿砖。」
3. 顶部有简单导航：仪表盘 / 软件管理 / 设置
4. 右上角有一个小提示入口「如何开始记录？」

### 关键 UI 要求
- 不要任何吓人的表单，一眼就知道「先去画画就行」
- 空状态要鼓励人——偏正向反馈，而不是"你还没画画哦"的内疚感

## 2）日常使用：每天画完之后看一眼自己有没有「断更」

### 场景
用户今天用 CSP 画完，睡前打开 RingoTrack。

### 步骤
1. 打开 App 默认落在仪表盘页面，顶部是「最近 30 天 · 合并视图」小绿砖墙
2. 用户一眼看到：
   - 今天那一格是不是空白 / 很浅 / 中等 / 很深
   - 最近有没有一长条连贯的砖，或者中间断了几天
3. 用户把鼠标移到今天那一格：
   - 弹出 tooltip：2025-01-01 · 总绘画时间：2 小时 15 分钟
4. 用户往下滚一点，看到「按软件分行视图（最近 7 天）」的迷你版：PS 一行、CSP 一行、Krita 一行

### 关键 UI 要求
- 仪表盘一打开就要给「一眼信息」：今天有没有画、最近有没有断更
- 交互动作极少：基本就是「打开 → 看→ 关」

## 3）回顾过去一个月 / 一年：理解自己创作习惯

### 场景
用户想知道自己 2025 年上半年到底有多勤奋。

### 步骤
1. 在仪表盘顶部时间范围切换器选择「最近 365 天」或「2025 年」
2. 主视图的小绿砖墙拉成全年 GitHub 风格，紧凑排布
3. 用户用鼠标拖动画布 / 滚轮缩放（v0.1 可先不做缩放，只做全年固定视图）
4. 点击某天（例如某天特别深色）：
   - 右侧 / 底部弹出「当天详情面板」：
     - 当天总绘画时间
     - 各软件时长列表和占比条形图（v0.1 可以先只做列表）
5. 用户再切到上方 Tab：按软件：
   - 看到每个软件一行小绿砖，轻松对比 "哪段时间主要用 PS，哪段主要用 CSP"

### 关键 UI 要求
- 年视图要紧凑但不压抑，重点是形状：一块连绵的砖墙 vs 断断续续
- 当天详情面板是可选展开，不把主视图填满信息

## 4）管理绘画软件列表：把「我常用的画图软件」告诉 RingoTrack

### 场景
用户发现有一个自己常用的软件没被统计到，想补上。

### 步骤
1. 用户切到顶部导航软件管理
2. 页面上方是已经识别的绘画软件列表：
   - 列：图标、展示名（可编辑）、AppId、分组（主力 / 辅助）、平台
3. 页面中间有一个大按钮「将当前前台应用加入为绘画软件」：
   - 用户保持 Krita 在前台 → 点这个按钮 → 列表里多出一条 Krita
4. 用户可以：
   - 编辑显示名称（比如改成「主力：CSP」）
   - 设置分组：主力 / 辅助

### 关键 UI 要求
- 软件管理是「可选增强」，不会阻塞使用
- 默认内置一批规则，让用户即使不管这页也能用起来

# App 页面结构 & 风格构想

## 1）整体信息架构（适配桌面 App）

### 顶部 AppBar
- 左侧：RingoTrack logo + 小标题（例如「See your drawing habits」）
- 中间：Tab 栏或二级导航：仪表盘 / 按软件 / 软件管理 / 设置

### 主内容区域
- **仪表盘**：
  - 顶部：时间范围切换器（7 天 / 30 天 / 365 天）
  - 中部：合并视图小绿砖墙（主角）
  - 下方：简化版「按软件」小绿砖 + 今天的总时长小卡片
- **按软件**：
  - 左侧：软件列表（可点击 filter）
  - 右侧：当前选中软件的小绿砖视图
- **软件管理**：
  - 列表 + 「将当前前台应用加入为绘画软件」按钮
- **设置**：
  - 预留：主题（浅色/深色）、数据导出、隐私提示等

## 2）视觉风格基调

### 色彩
- 背景：偏中性深灰或浅灰，避免抢走小绿砖的注意力
- 主色：一套绿色梯度（呼应 GitHub 小绿砖），可以稍微偏暖一点，感觉更「画画」

### 字体 & 文案
- 中文为主，英文软件名原样保留（PS / CSP / Krita 等）
- 文案尽量短、口语化：比如「今天有画画」「已经连续画了 5 天」

## 3）关键 UI 状态

### 空状态（无数据）
仪表盘中间是一堵灰色虚线砖墙轮廓 + 一条轻松的提示

### 低活跃状态
大部分砖很浅，少量深色点在某些周末

### 高活跃状态
连续一片深浅错落的绿色，很直观的「习惯已形成」反馈

### 加载中 / 错误
- 加载中：热力图区加简单 skeleton 占位
- 错误（例如无法访问本地数据）：顶部轻量提示条 + 按钮「重试」
